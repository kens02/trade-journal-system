# trade-journal-system 操作手順書

個人投資家(SBI証券利用者)向け株取引管理アプリの使い方をまとめたユーザー向けマニュアルです。開発者向けの技術仕様は `docs/trade-journal-spec-v0.5.md`(要件定義)・`.claude/commands/implement-p2.md`(実装計画)を参照してください。

## 1. このアプリでできること

- **取引記録**: SBI証券の取引を1件ずつ手入力、またはCSVファイルから一括取込
- **売買ルール管理**: 自分の売買ルールを文書化し、バージョン管理しながら各取引に紐付け
- **損益の自動計算**: 同一銘柄・同一口座区分内で買付と売却をFIFO(先入れ先出し)で自動マッチングし、実現損益を算出
- **トレードジャーナル**: 取引の振り返りメモを長文で記録し、感情タグ・自由タグを付けて後から検索
- **簡易集計**: ルール別の取引件数・金額・遵守評価の内訳を確認

## 2. 動作環境・起動方法

- 対応ブラウザ: **最新版のGoogle ChromeまたはMicrosoft Edgeを推奨**します。Safariでも閲覧・簡易的な入力はできますが、ブラウザの仕様上データが予告なく消去されるリスクがあるため、正式なデータ管理には使わないでください。
- データはすべてブラウザ内(IndexedDB)に保存されます。**サーバーには一切送信されません**。
- ローカルで動作確認する場合:
  ```
  npm run build && npx serve out
  ```
  表示されたURL(例: `http://localhost:3000`)にブラウザでアクセスします。
- 本番運用ではVercelにデプロイして常時アクセスする想定です。
- **同じブラウザ・同じ端末からのみアクセスしてください。** 別のブラウザやスマートフォンでは別のデータベースとして扱われ、データは共有されません(端末間の同期は「10. データのバックアップ」のJSONエクスポート/インポートで手動対応します)。

## 3. 画面構成

画面上部のヘッダーから各画面に移動できます。

| ナビゲーション | 画面 | 内容 |
|---|---|---|
| 取引 | 画面A `/trades` | 取引の入力・一覧・編集・削除、実現損益の確認 |
| ルール | 画面B `/rules` | 売買ルールの作成・改訂・廃止・削除 |
| 集計 | 画面C `/summary` | ルール別の簡易集計 |
| インポート | 画面D `/import` | SBI証券CSVの取込 |
| ジャーナル | 画面E `/journal` | トレードジャーナルの記録・検索 |

ヘッダー右上の「バックアップ(JSON)」ボタンは、どの画面からでもクリックできます(詳細は10章)。

## 4. 取引を記録する(画面A: 取引)

### 4.1 新規取引の入力

1. 「取引」画面上部のフォームに以下を入力します。
   - **約定日**: カレンダーから選択(初期値は本日)
   - **銘柄**: 検索ボックスに名前を入力すると登録済み銘柄の候補が表示されます。該当がなければ「+ 新規銘柄として登録」から、銘柄名・コード(任意)・商品種別(国内株式/米国株式/投資信託/ETF)・通貨を指定してその場で新規登録できます
   - **売買区分**: 買/売
   - **口座区分**: 特定/NISA(成長)/NISA(つみたて)/旧NISA
   - **数量**: 株数・口数
   - **単価**: 通貨に応じて小数桁が変わります(JPY: 小数第1位まで、USD: 小数第4位まで)
   - **受渡金額**: 数量×単価から自動で提案されますが、手数料等を反映したい場合は自由に上書きできます(一度手入力すると以後は自動更新されません)
   - **ルール紐付け(任意)**: 有効なルールの最新バージョンから選択すると「遵守評価」(遵守/一部逸脱/逸脱)の入力欄が現れます
   - **一言メモ**: 自由記述(任意)
2. 「登録」ボタンで保存します。

### 4.2 一覧の見方

- 約定日の新しい順に表示されます。
- **実現損益**列: 売却行にのみ表示されます。複数の買付にまたがってマッチした場合は合算値が表示され、正の値は緑、負の値は赤で色分けされます。買付側にまだ消化されていない数量が残る「未解消」の売却がある場合は ⚠未解消n と表示されます(詳細は8章)。
- 各行の「編集」でフォームに値が読み込まれ、修正して「更新」できます。「削除」は確認ダイアログの後に削除され、紐付いていたルールの遵守評価やFIFOマッチも一緒に削除・再計算されます。
- 売却行には「マッチ内訳」ボタンがあり、クリックするとどの買付から何株・いくらの損益でマッチしたかが展開表示されます(8章参照)。
- 「ジャーナル」リンクをクリックすると、その取引に紐付けたジャーナルエントリの作成画面(画面E)に移動します。

### 4.3 データの永続性

ブラウザをリロードしてもデータは消えません(IndexedDBに保存されているため)。ただし、ブラウザの「閲覧データを削除」などの操作を行うと消えるので、定期的なバックアップ(10章)を推奨します。

## 5. 売買ルールを管理する(画面B: ルール)

### 5.1 ルールの作成

「ルール」画面の「新規ルール作成」フォームで、ルール名と以下の5セクションを入力し「作成」します。これがversion 1になります。

- 概要
- エントリー条件
- イグジット条件
- 除外条件
- 資金管理

### 5.2 ルールの改訂(バージョン管理)

- 一覧の「改訂」ボタンから、最新バージョンの内容を引き継いだ状態で編集し、**改訂理由(必須)**を入力して保存すると新しいバージョンが作られます。
- 過去のバージョンは変更されません。「履歴」ボタンで各バージョンの内容と改訂理由を切り替えて確認できます。
- 取引に紐付けられるのは「有効」状態のルールの最新バージョンのみです。

### 5.3 廃止・削除

- 「廃止」ボタンでルールを廃止(retired)状態にできます。廃止したルールは取引フォームの選択肢からは消えますが、既に紐付けられた取引・集計にはそのまま残り続けます。廃止したルールは「再有効化」でいつでも有効に戻せます。
- 「削除」ボタンは**取引に一件も紐付いていないルールのみ**物理削除できます(確認ダイアログあり)。1件でも紐付きがあるとエラーメッセージが表示され、その場合は「廃止」を使ってください。

## 6. 集計を確認する(画面C: 集計)

「集計」画面には、ルールごと(「ルールなし」も1行として集計)に以下が表示されます。

- 件数(買/売)
- 通貨別金額(JPY・USD別)
- 遵守評価の内訳(遵守/一部逸脱/逸脱の件数)
- 「version別に展開」でルール内のバージョンごとの内訳も確認できます

※ ルール別の実現損益・勝率などの分析は今後(P3フェーズ)対応予定です。個別取引の実現損益は画面Aで確認できます。

## 7. SBI証券のCSVを取り込む(画面D: インポート)

### 7.1 対応しているファイル

SBI証券サイトからダウンロードできる以下3種類のCSVに対応しています(ファイル名のパターンで自動判定されます)。SBI証券サイト内の具体的なメニュー名・配置は改定される場合があるため、以下のファイル名を手がかりに、該当する取引履歴・ポートフォリオ関連のページを探してダウンロードしてください。

| ファイル | 用途 |
|---|---|
| 約定履歴照会CSV(`SaveFile_*.csv`) | 国内株式・ETF・投資信託の取引を一括登録 |
| 米国株式 約定履歴CSV(`PaymentRecords_*.csv`) | 米国株式・米国ETFの取引を一括登録(USD建て) |
| ポートフォリオCSV(`ポートフォリオ_YYYYMMDD.csv`) | 現在の保有状況を取り込み、取引記録との差異を検証 |

文字コード(Shift-JIS/UTF-8、BOM有無)は自動判定されるため、ダウンロードしたファイルをそのままアップロードして問題ありません。

### 7.2 取引CSV(約定履歴照会・米国株式)の取込手順

1. 画面上部の「CSVインポート(取引履歴)」エリアにファイルをドラッグ&ドロップ、またはクリックして選択します。ファイル種別(国内/米国)は自動判定されます。
2. **プレビュー**画面で解釈結果(先頭20行相当)を確認し、問題なければ「次へ」。
3. 銘柄が自動照合できなかった場合、**未照合銘柄の解決**画面が表示されます。既存銘柄に対応付けるか、新規銘柄として登録します。投資信託など略称の表記ゆれがある場合、一度対応付ければ次回以降は自動照合されます(エイリアスとして保存)。
4. 既存の取引と同一条件(約定日・銘柄・売買区分・数量・単価)の行があった場合、**重複の確認**画面が表示されます。行ごとに取込/スキップを選ぶか、「すべてスキップ」で一括スキップできます。
5. 「取込実行」で確定します。取込後は**取込結果サマリー**(取込/スキップ/エラー件数)と、過去の**取込履歴一覧**が表示されます。
6. ヘッダーの列名が想定と一致しないファイルを選ぶと、その時点で明示的なエラーメッセージが表示され、何も取り込まれません。

### 7.3 ポートフォリオCSVの取込手順

「CSVインポート(ポートフォリオ)」エリアも同様の手順(ファイル選択→プレビュー→取込実行)です。取込後は以下の2種類のチェック結果が表示されます。

- **合計ブロック突合**: CSV内の各セクション合計と、明細行から再計算した値が一致するかのチェック。不一致があれば警告が表示されます。
- **保有数量差異レポート**: これまで登録した取引記録から算出した保有数量と、CSV記載の保有数量を比較します。差異がある場合は、取引履歴の取り込み漏れが疑われるため、不足分の取引を追加登録してください。

## 8. 損益マッチング(FIFO)を確認・修正する

### 8.1 自動マッチングの仕組み

- 売却が登録されると、**同一銘柄・同一口座区分**の中で約定日が古い買付から順に自動的にマッチングされ、実現損益が計算されます(画面Aの「実現損益」列)。
- 口座区分が異なる買付・売却は絶対にマッチングされません(例: 特定口座の買付とNISA口座の売却は別扱い)。
- 取引を追加・編集・削除すると、影響を受ける銘柄・口座区分の組み合わせだけが自動的に再計算されます。
- 1件の売却が複数の買付にまたがる場合、数量の比率で損益が按分されます。

### 8.2 未解消の売却(買付不足)

保有していたはずの数量より多く売却しているように見える場合(例: CSVで直近2年分しか取り込んでおらず、それ以前に買った分を売却したケースなど)、画面Aの実現損益列に「⚠未解消n」と表示されます。

対処方法: その銘柄・口座区分の**期首残高相当の買付取引を手動で追加登録**してください。登録すると自動的に再マッチングが行われ、警告が解消されます。

### 8.3 マッチ内容の手動修正

- 売却行の「マッチ内訳」を開くと、その売却がどの買付から何株・いくらの損益でマッチしたかが一覧表示されます。
- 各マッチ行の「編集」から、マッチ先の買付と数量を変更できます(候補は同一銘柄・同一口座区分内の買付のみ)。変更すると「手動」マッチとして保存され、以降の自動再計算で上書きされません。
- 手動マッチは「手動マッチ解除」でいつでも取り消せます。解除すると、その分の数量は自動マッチングの対象に戻ります。
- 指定した数量が買付・売却の残数量を超える場合はエラーになり、保存されません。

## 9. トレードジャーナルをつける(画面E: ジャーナル)

### 9.1 エントリの作成

「ジャーナル」画面上部のフォームで、以下のいずれかを選びます。

- **取引に紐付ける**: プルダウンから対象の取引を選択します(画面Aの「ジャーナル」リンクから遷移した場合は自動的に選択済みになります)。日付は取引の約定日が自動的に使われます。
- **日付のみ指定**: カレンダーだけで日単位のエントリを作成します(特定の取引に紐付かない振り返りに使えます)。

本文には長文(数千字程度)を入力でき、タグ(9.2節)を選んでから「登録」します。

### 9.2 タグ

- **感情タグ**: 初期状態で「焦り」「確信」「迷い」「退屈」の4種類が用意されています。エントリ作成・編集画面のチェックボックスで複数選択できます。
- **自由タグ**: 「新規タグ名」欄に入力して「+ 追加」すると新しいタグが作られ、その場で選択状態になります。
- 画面下部の「タグ管理」から、既存タグの**改名**・**削除**ができます。削除すると、そのタグが付いていた全エントリからも自動的に外れます。

### 9.3 編集と自動保存

- 一覧の「編集」から既存エントリを開けます。
- 編集中は本文の入力が2秒間止まると自動保存されます(「自動保存中...」→「自動保存しました」と表示)。すぐに保存したい場合は「更新」ボタンでも保存できます。
- 「削除」で確認ダイアログの後にエントリを削除します。

### 9.4 検索

検索ボックスにキーワードを入力すると、本文またはタグ名に部分一致するエントリだけが一覧に表示されます。全角・半角の表記ゆれ(例: 半角カナと全角カナ)は自動的に吸収されます。

## 10. データのバックアップ

- ヘッダーの「バックアップ(JSON)」ボタンをクリックすると、その時点の全データ(取引・ルール・紐付け・appMeta等)が1つのJSONファイルとしてダウンロードされます。ファイル名は `trade-journal-backup-YYYYMMDD-HHmm.json` の形式です。
- **現時点ではJSONファイルからの復元(インポート)機能・最終バックアップ日時の表示や超過警告は未実装です**(今後のフェーズで対応予定)。ブラウザのデータ削除・端末変更に備えて、こまめに手動でバックアップファイルをダウンロードし、安全な場所に保管することを推奨します。
- 端末間(PC⇔スマートフォンなど)でのデータ共有は、現時点では自動同期の仕組みがないため、上記のJSONバックアップを手動でやり取りする運用を想定しています(自動復元は未実装のため、当面は目視での再入力または今後のリストア機能の実装を待つ形になります)。

## 11. よくある質問・注意事項

**Q. ダークモードで文字が読みにくい**
A. 本アプリはOSの設定に関わらず常にライトテーマで表示されます。ダークモード時に文字が読めなくなる問題への対応として、意図的にライト固定にしています。

**Q. 同じ取引を誤って2回登録してしまった**
A. 画面Aの「削除」で片方を削除してください。紐付いていたルール評価・FIFOマッチも連動して削除・再計算されます。

**Q. 投資信託の銘柄名が毎回「未照合」になる**
A. CSV取込時の「未照合銘柄の解決」画面で一度既存銘柄に対応付けると、その表記はエイリアスとして保存され、次回以降は自動照合されます。

**Q. ポートフォリオCSVを取り込んだら保有数量に差異が出た**
A. その銘柄・口座区分について、まだ取り込んでいない過去の取引がある可能性があります。約定履歴照会CSV(または米国株式CSV)を確認し、不足分を追加取込・手動登録してください。

**Q. Safariで使うとどうなる?**
A. 閲覧や軽微な入力は可能ですが、iOSのITP(Intelligent Tracking Prevention)によりブラウザ側でデータが予告なく削除されるリスクがあります。正式な記録用途はChrome/Edgeを使用し、Safari利用時はこまめにバックアップしてください。
